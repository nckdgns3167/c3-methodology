# ADR-0005: Mobile Edit Deferred to Phase 2

- Status: Accepted (2026-05-06)
- Constraints affected: B7
- Related features: F-013 (편집기는 데스크톱 전용), F-102 (Phase 2 부분 해제)

## Context

6개월 베타 일정 보호 + 솔로 + AI 위임 환경. 모든 기능을 Phase 1에 넣으면 일정 폭발. 모바일 편집 UX는 데스크톱과 본질적으로 다름 — 별도 설계·구현·테스트 필요.

베타 출시(2026-11-30) 우선순위 1: _작동하는 데스크톱 편집기 + 발행 흐름_.

## Decision

Phase 1 모바일 뷰포트에서 사용자 허용 행위 4가지로 제한:

(a) 자기 페이지 미리보기 (실제 배포 화면)
(b) 인라인 텍스트 수정 (오타 정정 수준 — 단일 텍스트 노드 in-place 편집만, 길이 제한)
(c) 공유 링크 복사
(d) 대시보드 조회 (방문자 수, 조회 섹션 등)

컴포넌트 추가/삭제, 드래그, 레이아웃 변경, 스타일·테마 편집은 모두 차단.

## Consequences

- Phase 1 일정 6개월 보호 — 모바일 편집기 UX 작업 0
- 사용자 일부 불편 수용 (편집은 데스크톱 필수)
- F-013 캔버스가 모바일 뷰포트 진입 차단 강제
- F-102에서 Phase 2 진입 시 부분 해제 (베타 사용자 피드백 보고 결정)

## Alternatives Considered

- **처음부터 모바일 동등**: 일정 12개월 — 부지원사업 마감(Phase 1 1년)과 정합 X
- **모바일 완전 차단**: 사용자 적대적 — 인라인 텍스트 수정·미리보기 정도는 필요
- **반응형 편집기 (자동)**: 시도 가능하나 솔로 편집기 UX 디자인 비용 큼
